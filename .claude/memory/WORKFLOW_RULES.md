# Правила работы — ОБЯЗАТЕЛЬНО ЧИТАТЬ

## Главное правило

**НИКОГДА не писать код без реальных данных с сервера и согласованного плана.**

Было несколько случаев когда код писался вслепую → создавал новые баги. Не повторять.

## Схема работы (ноутбук + сервер)

```
1. Пользователь описывает проблему
2. Ноутбук-Claude даёт готовый промт для сервера
3. Пользователь запускает его на сервере
4. Сервер диагностирует → возвращает: файл, строка, причина
5. Ноутбук пишет точный фикс
6. git push → сервер: git pull && nssm restart chimera_bot
```

## Git — ветка и деплой

- Репо: `gitkit11/Chimera-ai-v09`
- Ветка: **всегда `master`** — никаких других веток
- Ветка `chimera-v2` упразднена с 16.04.2026

**Ноутбук (начало каждой сессии):**
```bash

```

**После изменений:**
```bash
git push origin master
```

**Сервер (обновление):**
```bash
git pull origin master && C:\nssm\nssm-2.24\win64\nssm.exe restart chimera_bot
```
